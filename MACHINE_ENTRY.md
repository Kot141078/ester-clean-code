## Hard invariants

- By default, `c` orchestrates agents; agents do not define `c`.
- Agents are bounded runtime processes and tools invoked under `c`.
- Continuity, privilege holding, and stopping authority remain at the `c` layer.
- Model replacement or worker rotation does not by itself redefine the entity.
- Runtime proof-of-possibility does not replace the normative stack.
- Runtime proof-of-possibility does not outrank stronger doctrinal or evidence sources outside its scope.
- Local runtime proof or local validation does not by itself make this repo public-facing corpus history.
- Current corpus-state snapshot routing is control context only; it is not runtime proof, release authorization, or public-history completion.
- Coupling-map routing is co-review discipline only; it does not auto-authorize edits to runtime-adjacent public surfaces.
- Minimality-policy routing does not flatten bounded implementation-facing notes into generic corpus doctrine by convenience.
- Staging-map routing is bounded local planning context only; it does not turn runtime-proof surfaces into a publish command or standalone promotion state.
- Local promotion docket routing is bounded file-level planning context only; it does not turn runtime-proof adjacent surfaces into promotion-ready evidence or standalone public-history authorization.
- Question-to-surface triage routing is bounded reading guidance only; it does not let runtime-proof questions bypass stronger doctrinal, status, or public-safe surfaces.
- Control-stack completeness routing is sufficiency guidance only; it does not turn runtime-adjacent dirtiness or maintenance discomfort into proof that a new meta-layer is needed.
- If runtime-proof surfaces ever enter public-history promotion review, they should move only with their minimally coupled corpus-control bundle rather than as isolated proof fragments.
- If runtime-proof surfaces ever enter public-history promotion review, they should also respect the earlier corpus-control tranche logic rather than appearing before supporting orientation and control layers.
- If runtime-proof surfaces ever enter public-history cutover planning, they should stay inside a bounded candidate set rather than being packed into an overloaded first public step.
- If runtime-proof surfaces ever enter public history, the accompanying note should stay dry, scope-limited, and explicit that runtime proof is not whole-corpus closure or productization.
- If runtime-proof public history later needs correction, the correction language should stay bounded and typed rather than defaulting to product rollback, whole-corpus retraction, or theatrical failure wording.
- Public skeleton surfaces here are bounded implementation-facing material; they do not authorize importing private runtime trees or unsafe operational content into the public corpus.
- External handoff from this repo should use the bounded runtime-proof profile rather than pretending to cover the whole corpus.
